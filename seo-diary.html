Creating the SEO Diary Page

This SEO Diary page serves as both a documentation tool and a demonstration of the website's evolution. The idea behind this page was to create a transparent record of all SEO-related work, decisions, and outcomes for this portfolio project. It allows visitors to understand the optimization journey and serves as a reference for future improvements.

The page itself follows SEO best practices with proper meta tags, structured data (Blog schema), and semantic HTML. By documenting SEO activities chronologically, this page also adds unique content to the website, which helps with indexing and provides value to both search engines and visitors interested in SEO practices.

Approach: Transparent documentation of SEO work as both a learning tool and a showcase of best practices.

15.01.2026 – Website launch

Published static website on GitHub Pages. Configured robots.txt, sitemap.xml and Google Search Console.

Result: website online, indexing in progress.

16.01.2026 – Content & structure update

Added projects page, updated meta tags and improved internal linking.

Result: improved crawlability and content depth.

17.01.2026 – Website redesign and structure expansion

Updated the main page design using a Tailwind Plus hero template and unified the visual style across all pages. Reworked the Projects page, created SEO Diary and Contact pages, and ensured consistent navigation between all sections.

Updated meta tags (title, description, keywords), improved internal linking, and refreshed sitemap.xml to include all available pages.

Result: improved site structure, better crawlability, clearer content hierarchy. Indexing status monitored in Google Search Console.

18.01.2026 – Comprehensive SEO optimization across all pages

Conducted a full SEO audit and implemented missing SEO elements across all pages. Added Open Graph tags and Twitter Card meta tags to contact.html, projects.html, and seo-diary.html for improved social media sharing. Enhanced structured data with appropriate Schema.org types: ContactPage for contact.html and Blog for seo-diary.html.

Added keywords meta tags to contact.html and seo-diary.html to match the comprehensive approach used on other pages. Implemented robots meta tags on all pages for better crawl control. Added performance optimization tags (preconnect, dns-prefetch) to all pages for faster external resource loading.

Enhanced projects.html Schema.org data with detailed ItemList containing individual project entries as SoftwareApplication objects with descriptions, categories, and programming languages. Updated sitemap.xml to include lastmod dates for all URLs, improving search engine understanding of content freshness.

Result: Complete SEO consistency across all pages. All pages now have comprehensive meta tags, structured data, and social media optimization. Improved sitemap provides better indexing signals. Expected improvements in social media link previews and search engine understanding of page content.

19.01.2026 – SEO Diary and Blog split, new Blog page, navigation update

Split the combined SEO Diary/Blog page into two distinct sections: SEO Diary (this page) remains a chronological record of SEO actions and outcomes; Blog is a new standalone page for long-form articles on SEO, web development, and related topics.

Created blog.html with an in-depth article on SEO indexing: how search engines discover, crawl, render, and index pages; the role of sitemaps and robots.txt; Google Search Console and indexing monitoring; common indexing problems and solutions; and best practices for reliable indexing. The Blog is linked from the main navigation on all pages.

Updated navigation across index.html, projects.html, contact.html, seo-diary.html, and blog.html to include a Blog link. Adjusted the SEO Diary page title from "SEO Diary/Blog" to "SEO Diary" to reflect the split. Added blog.html to sitemap.xml with appropriate lastmod and priority.

Result: Clear separation between documentation (SEO Diary) and editorial content (Blog). Better content depth and topical focus for SEO; improved internal linking and crawl paths via site-wide Blog link. New indexing-focused article supports relevance for search queries around SEO and technical indexing.
